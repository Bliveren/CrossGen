# Image2Tools 开发总计划

## 1. 项目目标

做一个“只为 Image 2 服务”的轻量工具。

用户打开后只做四件事：
1. 填入 API Key
2. 输入提示词或上传参考图
3. 生成/编辑图片
4. 下载结果

## 2. 产品原则

- 默认简单：大多数参数都给自动值
- 生成优先：先拿到结果，再考虑精细调参
- 编辑优先：参考图、局部重绘、多图编辑是核心能力
- 可追溯：每次任务都能回看参数、耗时、输出文件
- 本地优先：密钥与历史尽量只保存在本机

## 3. MVP 范围

### 必做

- API Key 配置
- Base URL 配置
- 连接测试
- 文生图
- 图生图
- 局部重绘 / mask 编辑
- 生成中流式预览
- 图片下载
- 历史记录
- 任务重试与错误提示

### 暂不做

- 聊天对话系统
- 多账号轮询
- 云端同步
- 复杂模板市场
- 团队协作

## 4. 推荐交互模型

主界面采用“三段式”：

- 左侧：参数与账号配置
- 中间：大画布 + prompt + 执行按钮
- 右侧：历史缩略图与操作入口

默认只显示最常用项，高级项折叠：

- 尺寸
- 质量
- 输出格式
- 压缩率
- 生成数量
- 流式预览

## 5. 用户主路径

### 路径 A：纯文生图

1. 输入 API Key
2. 选择 `gpt-image-2`
3. 输入 prompt
4. 点击生成
5. 查看流式预览
6. 下载或保存到历史

### 路径 B：参考图编辑

1. 上传一张或多张参考图
2. 输入编辑 prompt
3. 调整尺寸/质量
4. 点击编辑
5. 比较前后效果
6. 下载结果

### 路径 C：局部重绘

1. 上传源图
2. 在画布上涂 mask
3. 输入改动说明
4. 执行局部编辑
5. 导出结果

## 8. 关键技术决策

- 默认走 OpenAI Image API
- 默认模型 `gpt-image-2`
- 只在未来版本考虑 Responses API 的对话式 image tool
- 默认输出格式优先 PNG
- 请求超时设置要高于复杂生成耗时，建议 180-300 秒
- `gpt-image-2` 当前不支持透明背景，UI 只提供 `auto` 和 `opaque`
- `gpt-image-2` 输出固定为 base64 图像数据，不依赖短期 URL
- `gpt-image-2` 支持 `png` / `jpeg` / `webp` 输出；`jpeg` / `webp` 支持 `output_compression`
- `gpt-image-2` 支持 `stream: true` 和 `partial_images: 0..3`
- `gpt-image-2` 支持灵活尺寸，宽高必须为 16 的倍数、比例不超过 3:1、最长边不超过 3840、总像素在 655360 到 8294400 之间
- mask 编辑要求源图和 mask 格式、尺寸一致且小于 50MB，mask 必须包含 alpha 通道

## 9. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 生成时间长 | 用户以为卡死 | 流式预览 + 长超时 + 进度态 |
| mask 格式不对 | 编辑失败 | 前端自动校验尺寸和 alpha 通道 |
| Key 泄露 | 安全风险 | 仅本地加密保存，日志脱敏 |
| 参数过多 | 上手困难 | 高级项折叠，默认值自动化 |
| 输出质量波动 | 用户不稳定 | 提供低/中/高三档预设 |

## 10. 成功标准

- 1 分钟内能完成首次出图的配置
- 生成与编辑都能在一个页面内完成
- 用户不需要理解 API 细节也能完成常见操作
- 历史与下载路径清晰可靠
