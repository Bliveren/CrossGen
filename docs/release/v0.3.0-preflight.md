# v0.3.0 Release Preflight

Status: preparation only. Do not treat this document as release evidence.

This checklist turns the `docs/plans/v0.3.0-plan.md` Phase 5 work into an
execution script for the release round. It can be updated before release
approval, but the actual version bump, update manifest, evidence ledger, tag,
and GitHub Release must wait for explicit user confirmation.

## Current Boundary

Already allowed before release approval:

- Keep README / README.zh-CN and `RELEASE_NOTES.md` in pending-release wording.
- Run local build and mock verifiers.
- Prepare manual acceptance and screenshot steps.
- Keep `docs/release/evidence.json` on the current published version.

Not allowed before release approval:

- Do not change `package.json` to `0.3.0`.
- Do not point `docs/updates/latest.json` at missing or unuploaded `v0.3.0`
  assets.
- Do not mark `docs/release/evidence.json` gates as passed without matching
  evidence.
- Do not create a `v0.3.0` tag or GitHub Release.
- Do not claim real API acceptance, Windows native validation, macOS signing,
  or Apple notarization without recorded evidence.

## Preflight Commands

Run from a clean `main` after syncing `origin/main`:

```bash
git fetch origin --prune
git checkout main
git pull --ff-only origin main
git status --short --branch
pnpm build
pnpm verify:mock-api
pnpm verify:mock-gemini-api
pnpm verify:mock-model-discovery
pnpm verify:release-evidence
```

Expected pre-release state:

- `main == origin/main`.
- Only explicitly preserved untracked files remain, if any:
  - `docs/plans/v0.3.0-multi-api-implementation.md`
  - `logo.png`
  - `logo/`
- `pnpm verify:release-evidence` still validates the current published
  evidence ledger, not a fabricated `v0.3.0` ledger.

## Manual Acceptance Script

Use redacted API access labels in notes. Do not record raw API keys, bearer
tokens, private base URLs, or local home-directory paths.

1. Start the app from current `main`.
2. Configure API access A with a GPT Image 2-capable OpenAI-compatible endpoint.
3. Run model discovery for API access A.
4. Confirm GPT Image 2 launch is available from discovered models.
5. Add API access B with a Gemini / Nano Banana-capable endpoint.
6. Run model discovery for API access B.
7. Confirm Nano Banana 3 launch is available from discovered models.
8. Switch between API access A and B.
9. Confirm prompt text, reference images, saved keys, and model choices are not
   silently cleared by switching.
10. Create a prompt template with tags.
11. Search/filter the template list and apply the template to the prompt.
12. Import at least one local reference image into Gallery.
13. Create a Gallery folder and move the imported image into that folder.
14. Add a successful history result into a selected Gallery folder.
15. Filter Gallery by folder, tag, and search together.
16. Use an `@` Gallery chip, a `#` hex color chip, and a `~` template chip in
   the prompt composer.
17. Remove one prompt chip and confirm the final submitted prompt excludes it.
18. Run a generation request.
19. Reuse the completed history item.
20. Download a result and open the output folder.
21. Delete a Gallery folder and confirm images move to Uncategorized instead of
   being deleted.
22. Delete a Gallery image and confirm history outputs remain intact.

Record the result as one of:

- `passed`
- `failed`
- `blocked`

For failures or blockers, record:

- OS and version.
- App commit SHA.
- Provider category only, for example `OpenAI-compatible` or `Gemini-compatible`.
- Redacted model IDs if sensitive.
- Exact visible error text.
- Whether the issue reproduces after app restart.

## Screenshot Capture List

Capture fresh `v0.3.0` screenshots only after the UI is in a clean demo state.
Do not capture API keys, private base URLs, private filenames, or user-local
paths.

Required screenshots:

- Main workspace with API access, launch model, prompt templates, parameters,
  preview, Recent jobs, and Gallery visible enough to show the v0.3.0 layout.
- API access manager with at least two saved API accesses and no visible keys.
- Prompt template manager with search, tags, and a reusable template.
- Gallery folder management with All / Uncategorized / a custom folder, counts,
  and image cards.
- Prompt composer showing `@`, `#`, and `~` chips without overflow.
- A completed history result with reuse, copy, download, add-to-Gallery, and
  delete controls.

Recommended output names:

- `docs/assets/v030-workspace.png`

## Release Evidence Notes

When release approval is granted and actual assets exist, update
`docs/release/evidence.json` only with verifiable data:

- exact commit SHA
- package version
- command output summary
- artifact filename
- public release URL
- SHA256
- byte size
- platform and architecture
- signing / notarization status
- Windows native verification mode

After evidence updates, run:

```bash
pnpm verify:release-evidence
pnpm verify:release-evidence -- --require-complete
```
