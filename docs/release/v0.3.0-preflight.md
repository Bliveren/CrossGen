## Manual Acceptance Script

Use redacted API access labels in notes. Do not record raw API keys, bearer
tokens, private base URLs, or local home-directory paths.

1. Start the app from current `main`.
2. Configure API access A with a GPT Image 2-capable OpenAI-compatible endpoint.
3. Run model discovery for API access A.
4. Confirm GPT Image 2 launch is available from discovered models.
5. Add API access B with a Gemini / Nano Banana-capable endpoint.
6. Run model discovery for API access B.
7. Confirm Nano Banana 3 launch is available from discovered models.
8. Switch between API access A and B.
9. Confirm prompt text, reference images, saved keys, and model choices are not
   silently cleared by switching.
10. Create a prompt template with tags.
11. Search/filter the template list and apply the template to the prompt.
12. Import at least one local reference image into Gallery.
13. Create a Gallery folder and move the imported image into that folder.
14. Add a successful history result into a selected Gallery folder.
15. Filter Gallery by folder, tag, and search together.
16. Use an `@` Gallery chip, a `#` hex color chip, and a `~` template chip in
   the prompt composer.
17. Remove one prompt chip and confirm the final submitted prompt excludes it.
18. Run a generation request.
19. Reuse the completed history item.
20. Download a result and open the output folder.
21. Delete a Gallery folder and confirm images move to Uncategorized instead of
   being deleted.
22. Delete a Gallery image and confirm history outputs remain intact.

Record the result as one of:

- `passed`
- `failed`
- `blocked`

For failures or blockers, record:

- OS and version.
- App commit SHA.
- Provider category only, for example `OpenAI-compatible` or `Gemini-compatible`.
- Redacted model IDs if sensitive.
- Exact visible error text.
- Whether the issue reproduces after app restart.

## Release Evidence Notes

When release approval is granted and actual assets exist, update
`docs/release/evidence.json` only with verifiable data:

- exact commit SHA
- package version
- command output summary
- artifact filename
- public release URL
- SHA256
- byte size
- platform and architecture
- signing / notarization status
- Windows native verification mode
- Linux CI run and package job URLs

After evidence updates, run:

```bash
pnpm verify:release-evidence
pnpm verify:release-evidence -- --require-complete
```
