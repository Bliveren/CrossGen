# CrossGen Media And Video Roadmap Foundation

> Updated: 2026-07-22
> Applies after v0.3.1 product-owner acceptance

## Decision

v0.4.0 should not be the first version that introduces media types, video route behavior, ffmpeg packaging, queue UI, and the user-facing video MVP at the same time.

The path to video is split into three preparation releases:

```text
v0.3.2  image task reliability and visibility
v0.3.3  agent access productization
v0.3.4  media foundation
v0.3.5  video technology Preview
v0.4.0  short-video dynamic asset MVP
```

## v0.3.1 Boundary

v0.3.1 may keep non-callable forward-compatible metadata that already exists, but it must not expose unimplemented product features.

Allowed:

- Queue status/stage metadata already needed by the image queue.
- Non-callable capability fields that clearly report video/GIF as unavailable.
- Defensive parsing that treats old image assets as image assets.

Not allowed:

- No video generation UI.
- No GIF generation UI.
- No `generate_video` MCP tool.
- No `export_gif` MCP tool.
- No automatic frame extraction or poster generation.
- No ffmpeg-static packaging.
- No local model download or GPU runtime manager.
- No silent local server or port assumption.

## v0.3.4 Media Foundation

v0.3.4 owns media data structure work without calling a real video API:

- `MediaKind = "image" | "video" | "animation"`.
- `OutputAsset = ImageAsset | VideoAsset | AnimationAsset`.
- Backward-compatible `ImageAsset` mapping for old History and Gallery entries.
- `VideoAsset` metadata: path, poster path, duration, fps, dimensions, MIME, hash, provider task metadata.
- `AnimationAsset` metadata: GIF/WebP/frames/sprite kind, frame count, fps, loop metadata, source video.
- Media-aware CLI/MCP serialization.
- Media-aware History and a `ResultViewer` boundary.
- Download fallback for non-previewable media.
- Explicit state and queue schema migration tests.
- Managed media root and file hygiene rules.

v0.3.4 does not expose a paid video generation entry.

## v0.3.5 Video Technology Preview

v0.3.5 validates video infrastructure behind Preview controls:

- One video route mock covering submit/poll/download.
- One small real-provider sample gate, with explicit cost confirmation.
- ffmpeg-static per-platform packaging spike.
- App signing/notarization validation with ffmpeg included.
- Poster extraction.
- One local postprocess action, preferably MP4 to GIF.
- Queue stages for remote poll, downloading, postprocessing, cancellation, timeout, interrupted recovery, and temporary-directory cleanup.
- Disk-space, MIME, size, redirect, and failed-download checks.

Preview features must not change image generation, image editing, Gallery, or History behavior.

## v0.4.0 MVP

v0.4.0 has one product promise:

```text
First-frame image -> short video -> local GIF -> export.
```

Keep:

- One provider or model family.
- One route strategy.
- First-frame image-to-video only.
- 3-6 second duration.
- Submit/poll/download recovery.
- Poster and History video preview.
- Explicit cost confirmation.
- One high-quality GIF export path.

Remove from the v0.4.0 first release:

- Multiple provider-native adapters.
- First/last-frame dual input.
- Hover scrubbing.
- Batch prompt/seed variants.
- MSE automatic loop detection.
- Advanced chroma-key removal.
- Sprite Sheet export.
- Deep video Gallery management.
- Full CLI/MCP video asset mutation.
- Model-specific advanced parameter panels.
- Multi-ratio ad workflow automation.

## Compatibility Rule

Old history and Gallery assets without `kind` behave exactly like image assets. Media-aware helpers must not change download, edit, Gallery thumbnail, tag, search, or folder behavior for existing images.
