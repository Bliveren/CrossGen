## Decision

v0.3.1 is a release-candidate hardening line for a reliable agent-ready image runtime. Its user promise is intentionally narrow:

```text
Desktop, CLI, and MCP share one reliable local queue for image tasks.
```

This version remains a release candidate until the product owner tests and approves the exact installable package. The public Latest release and `docs/updates/latest.json` must continue to point to v0.3.0 until that approval is recorded.

## Scope Freeze

v0.3.1 is frozen to release-blocking work only:

- Preserve existing desktop image generation, image-to-image, guided-region editing, partial streaming, Gallery/History reuse, image editing, save, and download flows.
- Keep desktop, CLI, and MCP generation on the same durable queue.
- Keep CLI/MCP as Preview contracts until v0.3.3.
- Keep the no-Node packaged CLI launcher and direct App-executable MCP entry working.
- Keep queue, Gallery mutation, image core, packaged CLI/MCP, and agent integration smoke coverage passing.
- Keep real-provider gates focused on the image operations claimed by v0.3.1.

Do not add new provider-native adapters, video fields beyond already-non-callable metadata, queue UI expansion, MCP tool expansion, visual redesign, or large model-catalog changes in v0.3.1.

## Boundaries

- No HTTP server and no local port.
- MCP uses stdio only.
- CLI/MCP default to `readonly`.
- Write and generate modes must be explicit.
- Paid generation requires `--yes` or MCP `confirm: true`.
- Generation always goes through the durable queue.
- Default concurrency is `1`; higher concurrency is explicit and bounded.
- API keys are resolved through keyring and never persisted to queue/stdout/MCP results.
- Local path disclosure requires explicit confirmation.
- v0.3.1 is image-only; video/GIF capabilities are not callable.
- No large provider adapter sweep in v0.3.1.

## Agent Contract

Required discovery:

- `crossgen --version --json`
- `crossgen doctor --agent --json`
- `crossgen mcp config --client codex|claude-code|cursor --mode readonly|write|generate --json`

Every CLI/MCP result includes `schemaVersion: 1`, `requestId`, optional `correlationId`, and either `data` or structured `error`.

Stable errors include `CONFIG_NOT_FOUND`, `API_KEY_MISSING`, `CONFIRMATION_REQUIRED`, `NO_LIVE_QUEUE_WORKER`, `CAPABILITY_UNSUPPORTED`, `UNSUPPORTED_IN_THIS_VERSION`, `RATE_LIMITED`, `PROVIDER_ERROR`, `LOCK_TIMEOUT`, `PATH_NOT_ALLOWED`, `ASSET_NOT_FOUND`, and `FOLDER_ALREADY_EXISTS`.

Write/generate operations accept `idempotencyKey` when repeated agent calls could duplicate cost or file mutation.

## Queue Semantics

Queue file: `crossgen-queue.v1.json`.

Rules:

- History is user-visible; queue is execution truth.
- Queue cannot depend on history retention.
- Stale `running` items recover to `interrupted`.
- `interrupted` never retries without explicit confirmation.
- A stale worker host record is pruned during host registration and the host list remains bounded.
- `remoteJobHandle`, if present in future work, prevents blind resubmit after crash.
- `stage` is secondary; `status` is the public coarse state.

Allowed worker hosts:

- Desktop Electron main.
- `CrossGen --mcp` with `CROSSGEN_MCP_MODE=generate`.
- `CrossGen --cli generate --wait` or explicit CLI worker.

`--async` can return a queue id only when a live worker host exists. Without one, it returns `NO_LIVE_QUEUE_WORKER`.

## Forward Roadmap

Each later version has one user promise:

| Version | User promise | Main contents |
| --- | --- | --- |
| `0.3.1` | Reliable agent-ready image runtime | Release-blocking fixes, image workflow regression, queue/CLI/MCP image task closure, exact-package product acceptance |
| `0.3.2` | Image task reliability and visibility | Minimal queue UI, visible cancel/retry/failure reasons, worker online/offline state, restart recovery, reference-image downsampling, timeout and route diagnostics |
| `0.3.3` | Agent access productization | CLI enable/disable, PATH detection, App-move repair, Codex/Claude Code/Cursor MCP UI, `doctor --agent`, permission and cost confirmation UI, stable job/error/asset contract |
| `0.3.4` | Media foundation | `OutputAsset` union, `VideoAsset`, `AnimationAsset`, media-aware History/ResultViewer/serialization, explicit state/queue migration, managed media root; no real video API |
| `0.3.5` | Video technology Preview | Video route mock, one small real API gate, ffmpeg-static packaging, poster extraction, one local postprocess action, cancellation/timeout/disk cleanup validation |
| `0.4.0` | Short-video dynamic asset MVP | One provider or model family, one route strategy, first-frame image-to-3-6s-video, submit/poll/download recovery, preview, cost confirmation, GIF export |

Model expansion, ComfyUI, local FLUX/SD, advanced batch variants, first/last-frame video, sprite sheets, chroma-key workflows, and deep video Gallery management do not belong to v0.3.1.

## Branch Hygiene

Release-blocking fixes must start from latest `origin/main`. Each report must include worktree path, branch, base SHA, ahead/behind, dirty status, validation commands, risks, and cleanup records.
