# CrossGen v0.3.0 - 多 API 支持功能实现进度

> 创建日期：2026-06-24  
> 目标版本：v0.3.0  
> 功能目标：支持用户配置、保存和切换多个 API 接入点

## 功能概述

用户可以配置多个不同的 API 接入（OpenAI、Gemini、Custom），每个接入保存独立的：
- API Key
- Base URL
- Provider 配置
- 模型发现结果
- 启动模型选择

用户可以随时在已配置的 API 之间切换，无需重新输入凭据。

## 验收标准

### 功能验收

- [ ] **添加 Provider**
  1. 点击"添加 Provider"按钮
  2. 选择 Provider 类型（OpenAI/Gemini/Custom）
  3. 输入 API Key 和 Base URL
  4. 保存后自动切换到新 Provider
  5. 新 Provider 出现在列表中

- [ ] **切换 Provider**
  1. 在下拉列表中选择另一个 Provider
  2. 服务配置区显示该 Provider 的配置
  3. 启动模型按钮状态根据该 Provider 的模型发现结果更新
  4. 参数面板保留当前 Provider 的参数设置

- [ ] **删除 Provider**
  1. 点击 Provider 条目的删除按钮
  2. 弹出确认对话框
  3. 确认后 Provider 从列表中移除
  4. 如果删除的是当前激活 Provider，自动切换到第一个
  5. 最后一个 Provider 无法删除

- [ ] **编辑 Provider 名称**
  1. 点击 Provider 名称进入编辑模式
  2. 修改名称后按回车或失焦保存
  3. 名称更新显示在选择器中

- [ ] **历史兼容**
  1. 旧的 v0.2.x state 文件能正常加载
  2. 单 config 自动迁移为 providers[0]
  3. 历史记录中的 providerId 能正确关联
